# 🗺️ KEYWORD MAPPING & ARQUITECTURA (KAKEBO 2026)
*Objetivo: Asignar 1 única intención de búsqueda por URL para eliminar la canibalización y enfocar el ranking orgánico.*

---

## 🏠 Nivel 1: Core pages (Producto y Marca)

### 1. Homepage (`/es`)
- **Intención:** Navegacional / Transaccional principal ("Busco la app").
- **Target:** Usuarios que buscan el producto por su nombre o buscan la mejor app de ahorro usando el método.
- **Palabra clave principal:** `kakebo ai`
- **Keywords secundarias:** `app kakebo`, `app de ahorro japones`, `app control de gastos`
- **Nota técnica SEO:** Esta página debe tener alta autoridad de dominio. Solo debe rankear para LA APP.

### 2. Página Tutorial / Cómo funciona (`/es/tutorial`)
- **Intención:** Informacional alta / Instruccional.
- **Target:** Usuarios curiosos que quieren saber el paso a paso del método dentro o fuera de la app.
- **Palabra clave principal:** `como hacer kakebo`
- **Keywords secundarias:** `metodo kakebo paso a paso`, `como aplicar kakebo`, `ejemplo metodo japones ahorro`
- **Acción:** No debe competir con la home tratando de vender la app en el H1. Debe ser un artículo tutorial honesto que, como conclusión, recomienda descargar Kakebo AI para automatizar el proceso.

## 🚫 Regla Estricta Anti-Canibalización a aplicar a partir de hoy:
1. **Nunca** uses la frase exacta "App de Kakebo" en el <title> de los artículos del blog (eso es exclusivo de la Home).
2. **Nunca** intentes explicar el método detallado en la home (para eso existirá la URL `/tutorial`).
3. Para cada nueva herramienta o post creado, revisa este mapa. Si la keyword ya está ocupada por otra URL, enlaza internamente en lugar de escribir contenido nuevo sobre el mismo tema.
